# STM32H723 双缓冲实时滤波实现说明

本文记录当前工程中已经跑通的 ADC -> IIR -> DAC 双缓冲实时滤波方案。代码主体位于 `Core/App`：

- `rlc_app.c`：应用状态机
- `rlc_display.c`：串口屏、USART 指令、曲线显示、调试串口
- `rlc_filter.c`：扫频、RLC 拟合、IIR 系数生成、实时滤波、ADC/DAC DMA 回调

## 1. 工作流程

系统上电后进入 `IDLE` 状态，等待串口屏或 USART 指令。

```text
IDLE
  | start_learn
  v
LEARNING
  | 扫频 + 拟合 + 生成 IIR 系数
  v
IDLE
  | start_filter
  v
FILTERING
  | start_learn
  v
停止实时滤波 -> LEARNING
```

关键点：

- 收到 `start_learn`：开始扫频学习。
- 收到 `start_filter`：进入实时滤波。
- 在 `FILTERING` 状态再次收到 `start_learn`：先停止 ADC/DAC/TIM DMA，再重新扫频学习。

## 2. 双缓冲参数

当前实时滤波参数：

```c
#define IIR_BLOCK_SAMPLES  (512U)
#define IIR_DMA_SAMPLES    (IIR_BLOCK_SAMPLES * 2U)
```

也就是：

- 半缓冲：`512` 点
- 全缓冲：`1024` 点
- ADC DMA buffer：`iir_adc_buf[1024]`
- DAC DMA buffer：`iir_dac_buf[1024]`
- 数据类型：`uint16_t`
- ADC 为 16-bit 数据，DAC 为 12-bit 右对齐输出

缓冲区定义为 32 字节对齐：

```c
#define ADC_DMA_ATTR __attribute__((aligned(32)))
#define DAC_DMA_ATTR __attribute__((aligned(32)))
```

这是为了配合 Cortex-M7 D-Cache 维护。

## 3. 实时滤波启动流程

实时滤波由 `rlc_filter_start_realtime()` 进入，内部调用 `iir_start_from_fit()`。

启动顺序：

1. 检查是否已经有拟合生成的 IIR 系数。
2. 清空 IIR 状态。
3. 初始化 ADC/DAC DMA buffer：
   - ADC buffer 初值：`32768`
   - DAC buffer 初值：`2048`
4. 如果 D-Cache 开启：
   - 清理 DAC buffer cache
   - 清理并失效 ADC buffer cache
5. 配置 ADC DMA 为 circular。
6. 配置 DAC DMA 为 circular。
7. 配置 TIM2 作为 ADC/DAC 触发时钟。
8. 启动 ADC DMA。
9. 启动 DAC DMA。
10. 启动 TIM2。

启动成功后 `iir_running = 1`，状态机进入 `FILTERING`。

## 4. 双缓冲中断与处理

ADC DMA 半满中断：

```c
iir_process_flags |= 0x01U;
```

ADC DMA 全满中断：

```c
iir_process_flags |= 0x02U;
```

主循环中 `rlc_filter_task()` 调用 `iir_process_pending()`，根据 flag 处理对应半块：

```text
flag 0x01 -> 处理 offset = 0
flag 0x02 -> 处理 offset = IIR_BLOCK_SAMPLES
```

真正的数据处理在：

```c
iir_process_block(offset)
```

这样 ADC DMA 可以继续采集另一半缓冲，CPU 同时处理当前半缓冲，形成双缓冲流水。

## 5. D-Cache 一致性处理

STM32H723 是 Cortex-M7，DMA 和 D-Cache 不自动一致。如果不维护 cache，会出现 ADC 数据读旧值、DAC DMA 读不到 CPU 写入结果的问题。

当前处理方式：

处理 ADC 半块前：

```c
SCB_InvalidateDCache_by_Addr((uint32_t *)&iir_adc_buf[offset],
                             IIR_BLOCK_SAMPLES * sizeof(iir_adc_buf[0]));
```

写完 DAC 半块后：

```c
SCB_CleanDCache_by_Addr((uint32_t *)&iir_dac_buf[offset],
                        IIR_BLOCK_SAMPLES * sizeof(iir_dac_buf[0]));
```

因此：

- CPU 读 ADC buffer 前，先让 cache 失效，确保读到 DMA 写入的新数据。
- CPU 写 DAC buffer 后，清理 cache，确保 DAC DMA 能读到新输出。

## 6. IIR 软件滤波路径

当前默认使用软件 IIR：

```c
#define IIR_USE_FMAC (0U)
```

每个半块处理流程：

1. 读取 ADC 半缓冲。
2. 将 ADC 采样转换为 `-1.0 ~ +1.0` 浮点输入。
3. 调用 CMSIS-DSP biquad：

```c
arm_biquad_cascade_df1_f32(&iir_biquad,
                           iir_float_buf,
                           iir_float_buf,
                           IIR_BLOCK_SAMPLES);
```

4. 将滤波结果转换为 DAC 码值。
5. 写入对应 DAC 半缓冲。
6. 清理 DAC buffer cache。

## 7. 输出幅值校准

当前最终实时输出幅值统一乘以 `0.8`：

```c
#define IIR_OUTPUT_GAIN (0.8f)
```

软件 IIR 输出路径：

```c
dac = 2048.0f + (value * 2047.0f * IIR_OUTPUT_GAIN);
```

FMAC 或直通调试路径使用：

```c
dac = 2048.0f + ((sample - 2048.0f) * IIR_OUTPUT_GAIN);
```

这个增益只作用于最终 DAC 输出，不影响扫频、拟合和 IIR 系数计算。

## 8. 停止实时滤波

当滤波状态下收到 `start_learn`，状态机会调用：

```c
rlc_filter_stop_realtime();
```

内部停止：

```c
HAL_TIM_Base_Stop(&htim2);
HAL_ADC_Stop_DMA(&hadc1);
HAL_DAC_Stop_DMA(&hdac1, DAC_CHANNEL_1);
iir_running = 0U;
iir_process_flags = 0U;
```

随后重新进入扫频学习流程。

## 9. 串口屏交互

USART3 接收串口屏命令：

- `start_learn`：开始扫频学习
- `start_filter`：开始实时滤波

USART1 用作调试串口，同时转发 USART3 收到的字节，便于观察串口屏实际发送内容。

扫频曲线发送到串口屏：

```text
add 3,0,<幅值>
add 3,1,<相位>
```

每条命令后追加：

```text
FF FF FF
```

曲线高度范围：

```text
0 ~ 127
```

曲线横向最多发送：

```text
320 点
```

## 10. 当前成功方案的核心原因

本方案能稳定输出，关键在于以下几点：

1. ADC DMA 和 DAC DMA 都使用 circular 模式。
2. ADC/DAC buffer 使用双半区：前 512 点、后 512 点。
3. 半满/全满中断只置位，不在中断里做重计算。
4. 主循环处理半块数据，避免中断里运行 IIR。
5. Cortex-M7 D-Cache 做了正确的 invalidate/clean。
6. DAC DMA 判断不再依赖 Priority，而是明确重新配置 circular。
7. 状态机允许在滤波中通过 `start_learn` 退出滤波并重新学习。

## 11. 后续可优化方向

- 将 DMA buffer 放入 MPU non-cacheable 区域，减少手动 cache 维护。
- 后续启用 FMAC 时，保留当前双缓冲框架，只替换半块滤波计算函数。
- 如果实时采样率继续提高，需要进一步压缩 `IIR_BLOCK_SAMPLES` 或减少浮点运算开销。
- 可以在串口屏增加状态显示，例如 `IDLE / LEARNING / FILTERING`。
